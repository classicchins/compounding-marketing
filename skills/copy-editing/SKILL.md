# Copy Editing & Improvement

Edit existing copy to improve clarity, persuasion, and conversion while maintaining brand voice.

## Process

### Step 1: Audit Copy Against Checklist
- [ ] Headline: Specific and benefit-clear?
- [ ] Subhead: Expands on headline?
- [ ] Benefits before features?
- [ ] Customer language (not jargon)?
- [ ] Active voice?
- [ ] Scannable (short sentences/paragraphs)?
- [ ] Social proof included?
- [ ] Strong CTA?
- [ ] Matches brand voice?

### Step 2: Apply Edits

**Common improvements:**
- **Vague → Specific:** "Fast" → "70% faster deployment"
- **Passive → Active:** "Projects can be tracked" → "Track every project"
- **Feature → Benefit:** "Real-time sync" → "Never lose work to version conflicts"
- **Generic → Differentiated:** "Best project management" → "Project management that syncs with your existing tools"

### Step 3: Output

```markdown
## [Section Name]

**Current:**
[Existing copy]

**Edited:**
[Improved copy]

**Changes:**
- [What changed and why]
```

## Quality Bar
- Every edit has clear rationale
- Maintains brand voice
- Improves clarity or persuasion
- Evidence-based (not just preference)
