# Paywall & Upgrade Optimization

Optimize moments where users hit limits and are prompted to upgrade.

## Process

### Step 1: Identify Trigger Points
- Feature limit (e.g., "You've reached 3 projects")
- Usage limit (e.g., "100 API calls used")
- Access barrier (e.g., "Premium feature")

### Step 2: Design Upgrade Moment
**Bad:**
"Upgrade to continue"

**Good:**
"You've created 3 projects! Upgrade to create unlimited projects and unlock [benefit]."

**Elements:**
- Acknowledge success (celebrate what they've done)
- Show value (what they unlock)
- Reduce friction (inline upgrade, no new page)

### Step 3: Pricing Display
- Show annual savings
- Highlight most popular plan
- Add trust signals (money-back guarantee)

### Step 4: Reduce Anxiety
- "Cancel anytime"
- "14-day money-back guarantee"
- "No credit card for trial"

## Output
Upgrade moment copy, pricing display recommendations, A/B test ideas.
