# The Compounding Philosophy

Why this plugin exists and how it's different from everything else.

## The Problem with Generic AI Prompts

Most AI marketing tools give you this:

```
Write a landing page for my SaaS product
```

**What you get:** Generic copy that could apply to any product. Features listed, no differentiation, no voice. AI slop.

**What's missing:**
- Your positioning (who you're for, how you're different)
- Your customer research (what language they use)
- Your brand voice (how you sound)
- Your messaging framework (proof points, objections)

**Result:** You edit for 2 hours to make it usable. Faster to write from scratch.

---

## The Compounding Marketing Approach

**Instead of:** "Write a landing page"

**We say:** "First, do the research. Then position. Then message. Then write."

```
1. Run /cm:research → ICP + competitive analysis
2. Run /cm:position → Positioning canvas + messaging
3. Run copywriting → Page copy (informed by 1+2)
```

**What you get:** Copy that's specific to your product, differentiated, on-brand, and backed by research.

**Time investment:**
- Upfront: 8-12 hours (research + positioning)
- Per page after: 1-2 hours (vs 3-4 hours from scratch)

**The compound effect:** Every subsequent page gets faster because positioning is done.

---

## Why "Compounding"?

Traditional marketing accumulates chaos:

- **Campaign 1:** Learn something, don't document it
- **Campaign 2:** Start from scratch, repeat mistakes
- **Campaign 3:** Still guessing, still inconsistent
- **Campaign 4:** Positioning drifts, messaging inconsistent

**Compounding marketing inverts this:**

- **Campaign 1:** Research → positioning → execute → document learnings
- **Campaign 2:** Reads learnings from Campaign 1 → builds on it → faster execution
- **Campaign 3:** Reads learnings from 1+2 → even faster
- **Campaign 4:** Marketing machine hums, compounding velocity

**The flywheel:**

```
Research → Positioning → Messaging → Copy → Feedback → Learnings
    ↑                                                       ↓
    ←←←←←←←←←← Compound Knowledge ←←←←←←←←←←←←←←←←←←←←←
```

Each project makes the next project easier.

## 80% Research, 20% Execution

**Traditional approach:**
- 10% research (quick Google search)
- 90% execution (write, rewrite, rewrite again)

**Compounding approach:**
- 80% research (ICP, competitive, positioning)
- 20% execution (copy flows from positioning)

**Why this works:**

Bad copy comes from unclear positioning.

If you don't know:
- Who you're for (ICP)
- How you're different (positioning)
- What proof you have (messaging)

...then writing copy is guessing. And guessing takes forever.

**Do the hard thinking upfront. Execution becomes easy.**

## Deliverables, Not Fluff

Every skill outputs a file to `deliverables/`.

**No "AI responses" that disappear.**  
**No "let me think about that" without documentation.**  
**Deliverables or it didn't happen.**

**Examples:**

- `positioning` skill → `deliverables/positioning-canvas.md`
- `icp-research` skill → `deliverables/icp.md`
- `copywriting` skill → `deliverables/homepage-copy.md`

**Why this matters:**

1. **Reusability** — Use positioning canvas for sales deck, ads, content
2. **Compounding** — Next skill reads previous deliverables
3. **Accountability** — No vague outputs, everything is specific
4. **Versioning** — Track changes over time (git-friendly)

## The Anti-Slop Commitment

**What we don't do:**

- ❌ Generic outputs ("Write a compelling landing page...")
- ❌ AI jargon (synergy, leverage, robust, seamless, cutting-edge)
- ❌ Vague recommendations ("Consider testing...")
- ❌ Unsourced claims ("Studies show...")
- ❌ One-size-fits-all templates

**What we do:**

- ✅ Specific outputs based on your product and research
- ✅ Plain English (short sentences, active voice)
- ✅ Actionable recommendations ("Test headline A vs B, measure CTR")
- ✅ Cited frameworks ("April Dunford's positioning canvas...")
- ✅ Customized to your context (reads `.cm-context` every time)

**Quality bar:**

Every skill output must:
1. Be specific to your product (not generic)
2. Include rationale (why this recommendation)
3. Cite sources or frameworks (not made up)
4. Be actionable (clear next steps)
5. Avoid AI slop (no jargon, no fluff)

## How This is Different from a Prompt Library

**Prompt libraries:**
- Collection of prompts
- No structure
- No context loading
- Output quality depends on how you write the prompt

**Compounding Marketing:**
- Structured workflows (skills chain together)
- Context-aware (reads `.cm-context` + `deliverables/`)
- Framework-driven (teaches you the methodology)
- Compounding knowledge (learnings accumulate)

**Prompt library:** "Here's 100 prompts. Good luck."

**Compounding Marketing:** "Here's a system. Research → position → execute → compound."
